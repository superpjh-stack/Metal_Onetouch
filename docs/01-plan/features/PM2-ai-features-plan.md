## 1. Overview

### 1.1 Purpose

원터치 금속 가공 제조 현장의 전 공정(입고→수주견적→공정→출하)에 AI/ML 기술을 내재화하여 품질 판단 자동화, 견적 산출 지능화, 공정 최적화, 설비 예방정비를 실현한다. 본 문서는 14개 AI 기능 각각의 입력/출력/알고리즘 및 상호 연결 구조를 정의한다.

### 1.2 Background

- 금속 가공 제조업의 특성: LOT 추적 기반 품질 관리, CAD 도면 기반 견적/BOM, 공정별 수율 관리
- 기존 MES는 데이터 수집 중심, AI 기반 판단 지원 부재
- 사업계획서 42-46페이지에 명시된 기능 범위를 기술 관점에서 구체화

### 1.3 Related Documents

- PDCA Status: `docs/.pdca-status.json`

## 3. AI 기능별 상세 설계

### 3.1 입고 AI Agent

| 항목 | 내용 |
|------|------|
| **입력** | 자연어 질의, LOT번호, 품목코드, 입고일자 |
| **출력** | 입고 품질 판단 결과, 관련 검사기준 문서 발췌, LOT 이력 요약 |
| **알고리즘** | RAG (Retrieval-Augmented Generation) |
| **모델** | Embedding: text-embedding-3-small (OpenAI) / 대안: BGE-M3, LLM: GPT-4o 또는 Claude 3.5 Sonnet |
| **지식베이스** | 입고기준서, 검사기준서, 과거 입고 이력(3년), 공급처 품질 이력 |
| **데이터 요구사항** | 입고기준서 PDF/Word 문서, MES 입고 이력 DB, 검사 결과 DB |
| **주요 기능** | 자연어로 "이 LOT 입고해도 되나요?" → 관련 기준 검색 후 판단 근거 제시 |

**처리 흐름:**
```
사용자 질의
    ↓
질의 임베딩 (text-embedding)
    ↓
벡터DB 검색 (top-k=5, 코사인 유사도)
    ├── 입고기준서 청크
    ├── 검사기준 청크
    └── 과거 LOT 이력
    ↓
컨텍스트 구성 (Retrieved Docs + LOT 메타데이터)
    ↓
LLM 프롬프트 생성 (시스템 프롬프트 + 컨텍스트 + 질의)
    ↓
응답 생성 (판단 결과 + 근거 문서 인용 + 신뢰도)
```

| **지식베이스** | 출하기준서, 품질기준서, 검사결과 DB, 클레임 이력(5년) |
| **데이터 요구사항** | 출하기준서 문서, 클레임 데이터 (고객사/품목/원인 구조화), 검사결과 테이블 |
| **주요 기능** | "LOT-2024-001 출하 리스크 확인" → 클레임 패턴 + 품질기준 비교 후 위험도 등급 제시 |

**리스크 등급 체계:**
```
GREEN  : 출하 승인 (검사 합격 + 유사 클레임 없음)
YELLOW : 주의 출하 (경미한 리스크 존재, 담당자 확인 필요)
RED    : 출하 보류 (검사 불합격 또는 반복 클레임 패턴 감지)
```

### 3.3 CAD도면 분석

| 항목 | 내용 |
|------|------|
| **입력** | CAD 도면 파일 (PDF, DWG, DXF) |
| **출력** | 구조화된 객체 목록 (홀/슬롯/형상/치수), JSON 형식 |
| **알고리즘** | PDF Parsing + Vision AI 객체감지 + OCR 치수 추출 |
| **모델** | PDF→이미지 변환: pdf2image, 객체감지: YOLOv8 (fine-tuned on 금속도면), 치수OCR: PaddleOCR, 통합판단: GPT-4o Vision |
| **데이터 요구사항** | 학습용 CAD 도면 최소 500장 (어노테이션 포함), DWG 파싱: ezdxf 라이브러리 |
| **주요 기능** | 도면 업로드 → 홀/슬롯 자동 감지 → 치수 추출 → 견적/BOM에 피처 전달 |

**출력 JSON 예시:**
```json
{
  "drawing_id": "DRW-2024-001",
  "objects": [
    {"type": "hole", "diameter": 12.5, "count": 4, "tolerance": "H7"},
    {"type": "slot", "width": 8.0, "length": 50.0, "count": 2},
    {"type": "bend", "angle": 90, "radius": 3.0, "count": 1}
  ],
  "dimensions": {"length": 200.0, "width": 150.0, "thickness": 3.2},
  "material_hint": "SUS304",
  "confidence": 0.91
}
```

### 3.4 객체인식 결과 관리

| 항목 | 내용 |
|------|------|
| **입력** | CAD 분석 결과 JSON + 원본 도면 이미지 |
| **출력** | 검증/수정된 결과 JSON, 학습 데이터셋 레이블 |
| **알고리즘** | 인터랙티브 어노테이션 UI (수동 수정) + 결과 이력 저장 |
| **모델** | Active Learning 파이프라인 (수정 데이터로 YOLO 재학습) |
| **데이터 요구사항** | 수정 이력 DB (원본 예측 vs 사람이 수정한 결과), 도면 이미지 스토리지 |
| **주요 기능** | 인식 결과 바운딩박스 시각화 → 작업자 수정 → 수정 이력 자동 학습데이터 적재 |

**Active Learning 사이클:**
```
모델 예측 → UI 표시 → 사람이 수정
    ↓
수정본 → 학습 데이터 풀 적재
    ↓
배치 재학습 (주 1회 또는 수정 건수 임계치 도달 시)
    ↓
신규 모델 성능 검증 (mAP 기준) → 배포
```

### 3.5 견적 자동산출

| 항목 | 내용 |
|------|------|
| **입력** | CAD 분석 결과 객체 피처 JSON, 재질, 수량, 납기 |
| **출력** | 공정별 단가, 총 견적금액, 과거 유사 견적 비교표 |
| **알고리즘** | Rule-based 공정별 원가 계산 + ML 보정 모델 |
| **모델** | 1단계: 규칙 기반 (피처 × 단가표), 2단계: XGBoost 보정 모델 (과거 견적 데이터 학습) |
| **데이터 요구사항** | 과거 견적 데이터 최소 1,000건 (피처 + 실제 원가), 공정별 단가표, 재질 단가 |
| **주요 기능** | 도면 업로드 → 자동 견적 산출 → 유사 견적 레퍼런스 비교 → 견적서 생성 |

**공정별 원가 계산 구조:**
```
총 견적 = 재료비 + Σ(공정비)
재료비 = 면적 × 두께 × 밀도 × 재질단가
공정비(절단) = 절단길이 × 절단단가/mm
공정비(홀가공) = 홀개수 × 직경보정계수 × 홀단가
공정비(절곡) = 절곡횟수 × 각도보정계수 × 절곡단가
공정비(용접) = 용접길이 × 용접단가/mm
ML 보정: XGBoost (입력: 전체 피처, 출력: 보정 계수)
```

### 3.8 영향요인 분석 (SHAP)

| 항목 | 내용 |
|------|------|
| **입력** | 학습된 ML 모델, 분석 대상 데이터 |
| **출력** | SHAP Summary Plot, Feature Importance 순위, 원가/품질 영향 변수 Top-10 |
| **알고리즘** | SHAP (SHapley Additive exPlanations) — TreeSHAP for XGBoost/LightGBM |
| **모델** | ML 분석(3.7)의 모델을 그대로 사용 |
| **데이터 요구사항** | ML 모델 결과물 + 원본 피처 데이터 |
| **주요 기능** | "이 견적 원가가 왜 높게 산출됐나?" → SHAP 기반 개별 예측 설명 제공 |

**출력 예시:**
```
원가 영향요인 분석 결과:
1. 재질(SUS304) : +15% 영향
2. 홀 직경(>10mm) : +8% 영향
3. 납기(3일 이내) : +12% 영향
4. 수량(<10개) : +6% 영향 (소량 생산 할증)
```

### 3.10 설비상태 모니터링

| 항목 | 내용 |
|------|------|
| **입력** | 설비별 센서 데이터 실시간 스트림 (전류, 온도, 진동, RPM) |
| **출력** | 설비 상태 등급, 이상 알림, 예측 고장 시점, 정비 권고 |
| **알고리즘** | 이상 감지: Isolation Forest + LSTM Autoencoder, 예측 정비: Survival Analysis |
| **모델** | 정상/이상 분류: Isolation Forest (비지도), 이상 패턴 학습: LSTM Autoencoder, 잔여 수명 예측: Weibull Regression |
| **데이터 요구사항** | IoT 센서 데이터 (샘플링: 1초~1분), 설비 정비 이력, 고장 이력 |
| **주요 기능** | 실시간 대시보드 → 이상 감지 시 즉시 알림 → 정비 스케줄 자동 제안 |

**이상 감지 임계값 설정:**
```
전류: 정상범위 ±2σ 초과 시 WARN, ±3σ 초과 시 ALERT
온도: 설정 상한 초과 시 ALERT
진동: 주파수 분석 (FFT) 기반 이상패턴 감지
RPM: 목표 RPM ±5% 벗어날 시 WARN
```

### 4.2 임베딩 전략

```
문서 유형별 청킹 전략:
┌─────────────────────────────────────────────────────────┐
│ 기준서/표준문서 (PDF)                                    │
│   → 섹션 단위 청킹 (max 512 tokens, overlap 50 tokens)  │
│   → 제목 메타데이터 포함                                 │
├─────────────────────────────────────────────────────────┤
│ LOT 이력 데이터 (구조화 DB)                              │
│   → LOT 단위 요약 텍스트 생성 후 임베딩                 │
│   → 원본 DB 레코드 ID를 메타데이터로 저장               │
├─────────────────────────────────────────────────────────┤
│ 클레임 이력 (비정형 텍스트)                              │
│   → 클레임 단위 청킹 (클레임번호, 내용, 원인, 조치)     │
│   → 날짜/고객사/품목 필터 메타데이터                    │
└─────────────────────────────────────────────────────────┘

임베딩 모델:
- 한국어 지원: BAAI/bge-m3 (다국어, 로컬 배포 가능)
- 대안 (API): text-embedding-3-small (OpenAI)
- 임베딩 차원: 1024 (BGE-M3) / 1536 (OpenAI)
```

### 4.3 검색 파이프라인

```
질의 수신
    ↓
질의 전처리 (한국어 형태소 분석, 용어 정규화)
    ↓
Hybrid Search
    ├── Dense Search: 임베딩 코사인 유사도 (top-k=10)
    └── Sparse Search: BM25 키워드 매칭 (top-k=10)
    ↓
RRF (Reciprocal Rank Fusion) 결합 (최종 top-k=5)
    ↓
메타데이터 필터 적용 (날짜, 공급처, 품목코드 등)
    ↓
Cross-encoder Re-ranking (ms-marco-MiniLM-L-6-v2)
    ↓
컨텍스트 구성 (최종 3-5개 청크)
    ↓
LLM 프롬프트 조립 → 응답 생성
```

## 5. Vision AI 파이프라인 (CAD 도면 분석)

### 5.1 전체 처리 흐름

```
도면 파일 업로드 (PDF/DWG/DXF)
    ↓
┌─ 파일 유형 분기 ─────────────────────────────────┐
│  DWG/DXF → ezdxf 파싱 → 벡터 객체 추출          │
│  PDF     → pdf2image → 고해상도 이미지 변환(300dpi)│
└───────────────────────────────────────────────────┘
    ↓
전처리 (이미지 정규화, 노이즈 제거, 이진화)
    ↓
객체 감지 (YOLOv8 fine-tuned)
    ├── 홀 (원형/타원형)
    ├── 슬롯 (직사각형 홀)
    ├── 외형 형상 (외곽선)
    ├── 절곡선
    └── 용접 기호
    ↓
치수 추출
    ├── OCR (PaddleOCR): 치수 수치 텍스트 인식
    ├── 치수선 감지: 화살표 방향 분석
    └── 치수-객체 매핑: 공간 근접성 기반 연결
    ↓
구조화 및 검증
    ├── 단위 추론 (mm 기본, inch 자동 변환)
    ├── 기하학적 일관성 검증 (치수 합산 등)
    └── 신뢰도 점수 계산
    ↓
JSON 구조화 출력 → 견적/BOM 모듈로 전달
    ↓
결과 시각화 (바운딩박스 오버레이) → 검토 UI 표시
```

### 5.2 YOLOv8 파인튜닝 전략

| 항목 | 내용 |
|------|------|
| **기반 모델** | YOLOv8n (nano) → YOLOv8m (medium, 정밀도 우선) |
| **학습 데이터** | 원터치 실제 도면 500+ (초기), 공개 기술 도면 데이터셋 보완 |

## 7. AI Agent 오케스트레이션

### 7.1 Agent 역할 분리

```
┌─────────────────────────────────────────────────────────────┐
│                    통합 AI 질의 (Router Agent)               │
│  - 사용자 의도 파악 → 적절한 Sub-Agent로 라우팅             │
│  - 복합 질의: 여러 Agent 결과를 조합하여 통합 응답 생성     │
└──────────────┬──────────────────┬───────────────────────────┘
               │                  │
    ┌──────────▼──────┐  ┌────────▼────────┐
    │  입고 AI Agent  │  │  출하 AI Agent  │
    │                 │  │                 │
    │ 지식베이스:      │  │ 지식베이스:      │
    │ - 입고기준서     │  │ - 출하기준서     │
    │ - 검사기준       │  │ - 품질기준서     │
    │ - 입고이력       │  │ - 클레임이력     │
    │                 │  │ - 검사결과       │
    │ 도구:           │  │                 │
    │ - LOT조회API    │  │ 도구:           │
    │ - 공급처DB조회   │  │ - LOT조회API    │
    └─────────────────┘  │ - 검사결과조회   │
                         │ - 클레임DB조회   │
                         └─────────────────┘
               │
    ┌──────────▼──────────────────────────────────┐
    │           공정/생산 분석 Agent              │
    │                                             │
    │ 도구: 공정실적DB, 품질DB, ML예측API,         │
    │       SHAP분석API, 설비센서DB               │
    └─────────────────────────────────────────────┘
```

### 7.2 Router Agent 라우팅 로직

```python
# 의도 분류 기준 (LLM 또는 규칙 기반)
routing_rules = {
    "입고 관련": ["입고", "납품", "공급처", "수입검사", "LOT입고"] → InboundAgent,
    "출하 관련": ["출하", "납기", "클레임", "출하검사", "리스크"] → OutboundAgent,
    "공정/생산": ["공정", "수율", "불량", "설비", "생산실적"]    → ProcessAgent,
    "견적/도면": ["견적", "도면", "CAD", "BOM", "원가"]         → EstimationAgent,
    "복합 질의": 여러 키워드 혼재                               → Multi-Agent 협력
}
```

### 7.3 Agent 간 협력 프로토콜

```
복합 질의 처리 예시:
"이번 달 클레임 난 LOT들의 공급처 입고 이력 분석해줘"

Step 1: Router → 출하 AI Agent (클레임 LOT 목록 조회)
Step 2: Router → 입고 AI Agent (해당 LOT 입고 이력 조회)
Step 3: Router → 공정 분석 Agent (공정 이상 여부 확인)
Step 4: Router 통합 응답 생성 (3개 Agent 결과 종합)

통신 프로토콜: 내부 API 호출 (동기) 또는 Message Queue (비동기, Celery)
응답 타임아웃: 단일 Agent 10초, 복합 질의 30초
```

### 7.4 사용자 질문 이력 활용

```
질문 이력 DB 스키마:
- user_id, session_id
- query_text, query_embedding
- agent_routed, response_text
- feedback_score (thumbs up/down)
- timestamp

활용 방안:
1. 유사 질의 캐싱 (임베딩 유사도 0.95 이상 시 캐시 응답)
2. 자주 묻는 질문 패턴 → 프롬프트 개선 인사이트
3. 낮은 평점 응답 → 학습 데이터 개선 우선순위
```

| 설비 모니터링 | 오경보율 (False Positive) | ≤ 5% | 알림 이력 분석 |

| 통합 AI 질의 | 응답 시간 | ≤ 8초 | APM 모니터링 |

### 8.2 RAG 시스템 품질 기준

| 지표 | 도구 | 목표값 |
|------|------|--------|
| Faithfulness (환각 방지) | RAGAS | ≥ 0.85 |
| Answer Relevancy | RAGAS | ≥ 0.80 |
| Context Recall | RAGAS | ≥ 0.78 |
| Context Precision | RAGAS | ≥ 0.75 |

### 8.3 시스템 전체 비기능 요건

| 항목 | 기준 |
|------|------|
| Agent 응답 시간 (단순 질의) | ≤ 5초 |
| Agent 응답 시간 (복합 질의) | ≤ 30초 |
| ML 모델 추론 응답 | ≤ 2초 |
| CAD 도면 분석 (A4 기준) | ≤ 30초 |
| 시스템 가용성 | 99.5% (월 기준) |
| 동시 사용자 | 50명 |
| 벡터DB 검색 레이턴시 | ≤ 100ms (top-5) |

### 8.4 모델 드리프트 모니터링

```
모니터링 주기: 주 1회 자동 리포트
드리프트 감지: PSI (Population Stability Index) > 0.2 → 경고
              PSI > 0.25 → 자동 재학습 트리거
품질 게이트: 재학습 후 KPI 달성 시에만 프로덕션 배포
             미달 시 이전 버전 유지 + 알림 발송
```

| MLOps | MLflow + Airflow | 오픈소스, 온프레미스 |
| 비전 AI | YOLOv8 + PaddleOCR | 경량, 파인튜닝 용이 |
| API 서버 | FastAPI (Python) | AI/ML 생태계 친화 |
| 백엔드 | Node.js / Python 혼용 | MES 연동 유연성 |
