## 1. Overview

### 1.1 Purpose

금속 가공(판금/용접/절삭) 제조 현장의 생산성 향상, 품질 불량 감소, 원가 예측 고도화를 위해
AI 기술(RAG, Vision AI, ML)을 MES 핵심 업무(수주→생산→품질→출하)에 통합하는 시스템을 구축한다.

### 1.2 Background

- 수작업 견적 산출로 인한 납기 지연 및 오산정 발생
- LOT 단위 품질 추적 부재로 클레임 원인 분석 불가
- IoT 설비 데이터 미활용으로 예방 정비 불가
- ERP와 MES 간 데이터 사일로로 경영 의사결정 지연

### 1.3 Related Documents

- 사업계획서 42-46페이지 (10대 모듈 기능 정의)
- 본 문서 기반 설계: `docs/02-design/features/mes-architecture.design.md` (예정)

### 2.2 Frontend Layer 상세

| 구성 요소 | 기술 | 역할 |

| 폼 처리 | React-Hook-Form + Zod | 입고/수주/공정 입력 폼 검증 |

## 3. 10대 모듈 우선순위 (Phase 분류)

### MoSCoW 분류 기준

| 우선순위 | 분류 기준 | 해당 모듈 |
|----------|-----------|-----------|
| Must | 제조 운영 핵심, 즉시 ROI 발생 | 공정관리, 기준정보관리, 사용자/시스템관리, AI 대시보드 |
| Should | 업무 효율화, 단기 ROI | 입고재고관리, 출하물류관리, KPI관리 |
| Could | AI 차별화, 중기 ROI | 수주견적AI관리, 데이터허브관리 |
| Won't (현재) | 복잡도 高, 장기 과제 | AI Agent 통합관리 (독립 Phase 3) |

---

### Phase 1: MES 핵심 기반 (Month 1~4, 16주)

**목표**: 생산 현장 즉시 투입 가능한 MES 기본 기능 완성

| 모듈 | 포함 기능 | 개발 기간 | 우선순위 |
|------|-----------|-----------|----------|
| 사용자/시스템관리 | RBAC, 로그관리, 알림설정, 시스템설정 | 3주 | Must |
| 기준정보관리 | 품질기준, 작업표준, 코드관리 | 2주 | Must |
| 공정관리 | 공정실적, 데이터 모니터링, 작업조건, 이력조회 | 5주 | Must |
| AI 대시보드 (기본) | 생산현황분석, 설비상태 모니터링 | 3주 | Must |
| IoT 연동 기반 | MQTT 수신, InfluxDB 적재, 실시간 표시 | 3주 | Must |

**Phase 1 완료 기준**:
- LOT 기반 공정 실적 등록/조회 정상 동작
- IoT 센서 4종(전류, 온도, 진동, RPM) 실시간 대시보드 표시
- RBAC 권한 체계 (생산관리자/품질담당자/공정기사/경영진) 적용
- 시스템 응답시간 평균 500ms 이내

**Phase 1 선택 이유**:
현장 투입 없이는 AI 학습 데이터 자체가 쌓이지 않는다. 공정관리와 IoT 연동이 모든 AI 기능의 데이터 원천이므로 최우선 구축한다.

---

### Phase 2: 물류/품질 + AI 기초 (Month 5~8, 16주)

**목표**: 입출고 추적, 품질 분석, 기초 AI 기능 완성

| 모듈 | 포함 기능 | 개발 기간 | 우선순위 |
|------|-----------|-----------|----------|
| 입고재고관리 | 입고관리, 원자재이력, 데이터관리, 공급처품질분석 | 4주 | Should |
| 출하물류관리 | 출하관리, LOT추적, 검사결과, 클레임분석 | 4주 | Should |
| KPI관리 | 생산성KPI, 품질KPI, KPI 목표설정 | 2주 | Should |
| AI 대시보드 (완성) | 품질현황분석, 출하현황분석 추가 | 2주 | Should |
| 입고/출하 AI Agent (RAG 기초) | 입고 이상 알림, 출하 클레임 분석 | 4주 | Should |

**Phase 2 완료 기준**:
- LOT 번호로 원자재 입고→공정→출하 전 구간 추적 가능
- 클레임 발생 시 원인 LOT 5분 이내 특정
- RAG Agent가 입고/출하 관련 자연어 질문에 응답
- KPI 대시보드 경영진 화면 실시간 갱신

**Phase 2 선택 이유**:
Phase 1에서 축적된 공정 데이터를 활용하여 전후 물류와 연결한다. RAG Agent는 기존 문서(작업표준, 품질기준)를 학습 데이터로 즉시 활용 가능하여 Phase 2 도입이 적절하다.

---

### Phase 3: AI 고도화 + 통합 (Month 9~14, 24주)

**목표**: AI 차별화 기능 완성 및 전체 시스템 통합

| 모듈 | 포함 기능 | 개발 기간 | 우선순위 |
|------|-----------|-----------|----------|
| 수주견적AI관리 | CAD도면분석, Vision AI 객체인식, 자동견적, BOM자동생성, SHAP분석 | 10주 | Could |
| 데이터허브관리 | 데이터통합, 조회, 시각화, 다운로드, AI학습데이터관리 | 4주 | Could |
| AI Agent 통합관리 | 통합AI질의, 생산/품질분석, 의사결정지원, 알림/추천, 질문이력 | 6주 | Could |
| ERP 완전 통합 | 양방향 데이터 동기화, 수주→BOM→공정 자동 연계 | 4주 | Could |

**Phase 3 완료 기준**:
- DWG/PDF 도면 업로드 후 10분 이내 견적 초안 자동 생성
- SHAP 기반 원가 영향요인 상위 5개 자동 표시
- 통합 AI Agent가 생산/품질/출하 전 도메인 질의 응답
- ERP 수주 데이터 실시간 MES 반영 (지연 30초 이내)

**Phase 3 선택 이유**:
수주견적AI는 Vision AI 모델 학습에 최소 6개월 분 도면 데이터가 필요하다. Phase 1~2에서 축적된 실제 생산 데이터가 ML 모델 정확도를 높이므로 Phase 3가 적합하다.

## 4. 핵심 사용자 스토리

### 역할 정의

| 역할 | 주요 업무 | 주요 사용 모듈 |
|------|-----------|----------------|
| 생산관리자 | 생산 계획, 실적 집계, 인력/설비 배분 | 공정관리, AI대시보드, KPI관리 |
| 품질담당자 | 입고 검사, 공정 품질, 출하 검사, 클레임 대응 | 입고재고관리, 출하물류관리, 기준정보관리 |
| 공정기사 | 설비 작업조건 설정, 이상 감지, 공정 데이터 기록 | 공정관리, IoT대시보드, AI Agent |
| 경영진 | 경영 현황 파악, 의사결정, KPI 목표 설정 | AI대시보드, KPI관리, AI Agent 통합관리 |
| 영업담당자 | 수주 접수, 견적 산출, 납기 확인 | 수주견적AI관리, 데이터허브관리 |

### US-01 (생산관리자) - 실시간 생산 현황 모니터링
```
As a 생산관리자,
I want to 전체 라인의 생산 실적(목표 대비 달성률, 불량률)을 한 화면에서 실시간으로 볼 수 있기를,
So that 라인별 이상 상황을 즉시 파악하고 인력을 재배치할 수 있다.

Acceptance Criteria:
- 라인별 금일 목표수량, 실적수량, 달성률이 5초 간격으로 갱신된다
- 불량률이 기준치(3%) 초과 시 해당 라인이 빨간색으로 강조된다
- 클릭하면 해당 라인의 LOT별 상세 실적으로 드릴다운된다
```

### US-02 (생산관리자) - LOT 기반 공정 이력 조회
```
As a 생산관리자,
I want to LOT 번호 하나로 해당 LOT의 전체 공정 이력(투입 원자재, 작업자, 설비, 검사 결과)을 조회하기를,
So that 품질 문제 발생 시 원인 공정과 담당자를 5분 이내에 특정할 수 있다.

Acceptance Criteria:
- LOT 번호 검색 후 3초 이내에 전체 이력 표시
- 원자재 입고 → 각 공정(순서대로) → 출하까지 타임라인 형태로 표시
- 각 공정에서 측정된 품질 수치와 합부 판정 포함
```

### US-03 (생산관리자) - 설비 가동률 KPI 관리
```
As a 생산관리자,
I want to 설비별 가동률, OEE(종합설비효율)을 주간/월간 단위로 집계하고 목표와 비교하기를,
So that 설비 투자 및 예방 정비 계획 수립에 활용할 수 있다.

Acceptance Criteria:
- OEE = 가용률 × 성능률 × 품질률 공식으로 자동 계산
- 전월 대비 추이 차트 제공
- KPI 목표값 설정 가능, 미달 시 알림 발송
```

### US-04 (품질담당자) - 입고 자재 품질 검사 등록
```
As a 품질담당자,
I want to 입고된 원자재의 검사 항목(치수, 경도, 성분 등)을 모바일에서도 입력하고 합부 판정을 기록하기를,
So that 불합격 자재가 공정에 투입되는 것을 원천 차단할 수 있다.
```

## 6. 통합 포인트 (ERP/MES/IoT 연동)

### 6.1 ERP 연동 방식

```
ERP 시스템 (SAP/더존/그룹웨어)
        │
        │  REST API / DB Link / File 인터페이스
        ▼
┌───────────────────────────────────────┐
│         ERP Adapter (FastAPI)         │
│                                       │
│  수신 데이터 (ERP → MES):            │
│  - 수주 정보 (품번, 수량, 납기)       │
│  - 원자재 발주 정보                   │
│  - 거래처 마스터                      │
│                                       │
│  송신 데이터 (MES → ERP):            │
│  - 생산 실적 (품번별 완성 수량)       │
│  - 원자재 소요/소진 실적              │
│  - 품질 불량 실적                     │
│  - 완성품 재고 현황                   │
└───────────────────────────────────────┘
        │
        ▼
   MES PostgreSQL

연동 방식 우선순위:
1순위: REST API 실시간 연동 (ERP가 API 제공 시)
2순위: 데이터베이스 링크 (동일 네트워크 내)
3순위: 파일 배치 연동 (CSV/Excel, 1시간 주기)
```

### 6.2 IoT 설비 연동 방식

```
설비 (PLC/CNC/프레스)
        │
        │  OPC-UA / Modbus / Serial
        ▼
┌─────────────────────────┐
│    IoT Edge Gateway     │  ← 현장 설치 (라즈베리파이 or 산업용 PC)
│  - 프로토콜 변환        │
│  - 데이터 전처리         │
│  - 로컬 버퍼링           │
└────────────┬────────────┘
             │  MQTT over TLS
             ▼
┌─────────────────────────┐
│   MQTT Broker           │
│   (Eclipse Mosquitto)   │
│                         │
│   Topic 구조:           │
│   onetouch/{site}/      │
│   {machine_id}/         │
│   {sensor_type}         │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│   Kafka Consumer        │  ← IoT 이벤트 스트림 처리
│   - 이상값 감지          │
│   - InfluxDB 적재        │
│   - 알림 트리거          │
└─────────────────────────┘
             │
        ┌────┴────┐
        ▼         ▼
  InfluxDB    Redis Pub/Sub
  (원시데이터  (실시간 알림)
   저장)

수집 데이터:
- 전류 (A): 설비 부하 및 이상 감지
- 온도 (°C): 과열 방지, 공정 품질
- 진동 (g): 베어링 이상, 툴 마모
- RPM: 스핀들 속도, 가공 조건
```

### 6.3 MES 내부 모듈 간 연동

```
수주견적AI관리
    │ 수주 확정 → BOM 자동 생성
    ▼
기준정보관리 (BOM/공정순서)
    │ 공정 기준 제공
    ▼
공정관리 (작업지시 발행)
    │ 실적 등록 → LOT 이력 기록
    ├──────────────────────────┐
    ▼                          ▼
입고재고관리                 품질관리 (공정검사)
(원자재 소진 처리)             │
                               ▼
                          출하물류관리 (출하검사 → 출하 승인)
                               │
                               ▼
                          ERP 연동 (실적 송신)
```

### 6.4 AI Agent 연동 포인트

| AI Agent | 데이터 소스 | 연동 방식 | 주요 기능 |
|----------|-------------|-----------|-----------|
| 입고 AI Agent | 입고 이력, 공급처 품질DB, 작업표준 | RAG (pgvector) | 입고 이상 감지, 공급처 품질 Q&A |
| 출하 AI Agent | LOT 이력, 클레임DB, 검사 기준 | RAG (pgvector) | 클레임 원인 분석, 유사 불량 검색 |
| 수주견적 AI | CAD 파일, 과거 수주 원가DB | Vision AI + ML | 도면 분석, BOM 생성, 원가 예측 |
| 통합 AI Agent | 전 모듈 DB + 벡터DB | RAG + Tool Calling | 범용 질의응답, 의사결정 지원 |
